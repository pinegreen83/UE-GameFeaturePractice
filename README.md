UE-GameFeaturePractice

Unreal Engine 5.4 환경에서 게임 기능 단위 설계·구현 역량을 실무 기준으로 훈련하기 위한 개인 연습 레포입니다.

이 레포의 핵심 목적은 하나의 완성된 게임을 만드는 것이 아니라,
“기능 하나를 어떻게 분석하고, 어떤 구조로 설계하며, 어떤 기준으로 검증하는가”에 대한 사고 과정을 반복적으로 축적하는 데 있습니다.

⸻

Why This Repo Exists

개인 프로젝트를 진행하며 자주 겪었던 문제는 다음과 같았습니다.
- 기획 범위가 커질수록 프로젝트를 시작하지 못함
- 시작하더라도 구조가 불안정해 중간에 방향을 잃음
- 무엇을, 어떤 기준으로 연습해야 하는지 명확하지 않음

이 레포는 이러한 문제를 해결하기 위해 다음 전략을 채택합니다.
- 대형 프로젝트 대신, 단일 기능 단위 훈련
- 티켓 기반으로 요구사항을 명확히 제한
- 설계 → 구현 → 검증 → 회고의 반복

즉, 이 레포는 *“작게 만들되, 실무 기준으로 깊게 고민하는 연습”*을 목표로 합니다.

⸻

Training Philosophy

이 레포에서의 모든 기능 구현은 다음 질문에 답할 수 있어야 합니다.
- 이 기능은 왜 필요한가?
- 책임은 어디까지 분리되어야 하는가?
- Unreal Engine 구조상 어떤 선택이 안전한가?
- 지금은 단순하지만, 확장 시 문제가 생길 여지는 없는가?

따라서 “동작하는 코드”보다 “설명 가능한 구조”를 더 중요하게 다룹니다.

또한 AI를 단순한 코드 생성 도구가 아닌,
- 설계 검토자
- 구조적 리스크를 지적하는 파트너
- 검증 관점 제공자
로 활용하는 협업 훈련을 함께 포함합니다.

⸻

Workflow

본 레포는 다음 사이클을 반복합니다.
1.	티켓 발행
- 기능 요구사항이 명확히 정의된 구현 티켓 생성
2.	설계 작성
- 책임 분리, 구조 선택, 확장 방향을 문서로 정리
3.	설계 검토
- Unreal Best Practice, 메모리/GC, Tick, 네트워크 관점에서 검증
4.	구현 및 테스트
- Unreal Engine 5.4 환경에서 실제 동작 확인
5.	검증 및 회고
- 문제점, 개선 포인트, 재사용 가능한 규칙 정리

이 과정을 통해 기능 하나가 단순한 구현 경험이 아닌, 재사용 가능한 사고 자산으로 남도록 합니다.

⸻

What This Repo Trains

이 레포를 통해 다음 역량을 훈련합니다.
- 기능 단위 요구사항 해석 능력
- Unreal Engine 기반 구조 설계 능력
- 메모리 / 포인터 / GC 안전성에 대한 감각
- Tick 사용 및 성능 고려 습관
- 네트워크 확장 가능성에 대한 선제적 판단
- 문제 원인 분석 및 구조 개선 능력

⸻

How to Use
1.	Progress_Log.md를 최신 상태로 유지
2.	티켓 발행 후 Docs/Days/DayXXX_...에 설계·구현·회고 기록
3.	구현 및 검증은 Sandbox/ Unreal 프로젝트에서 수행

⸻

Folder Guide
- Docs/Templates/ : 티켓 / 설계 / 회고 문서 템플릿
- Docs/Days/ : 일자별 티켓 수행 기록
- Sandbox/ : Unreal Engine 5.4 테스트 프로젝트
- Progress_Log.md : 전체 티켓 진행 상태 인덱스

⸻

Conventions
- Day 폴더명: Day###_<CATEGORY>-<ID>
- 티켓 ID 예시: INPUT-001, UI-002, NET-003

⸻

Notes
- 이 레포는 학습용 샘플 코드 모음이 아닙니다.
- 각 티켓은 실무에서 발생할 수 있는 문제 상황을 가정하여 설계됩니다.
- 결과물보다 설계 판단 근거와 문제 해결 과정을 더 중요하게 기록합니다.

이 레포는 “무엇을 만들었는가”보다,“어떤 기준으로 판단하고 구현했는가”를 보여주기 위한 기록입니다.
