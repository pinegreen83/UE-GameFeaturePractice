## What Worked
- 서버 권한 기반 상호작용 구조가 리슨 서버 환경에서 명확히 검증됨
- 힐킷 픽업 시 서버/클라이언트가 항상 동일한 월드 상태로 수렴
- “UI는 클라, 상태는 서버” 규칙이 코드 구조로 자연스럽게 강제됨

## What Broke
- 초기에는 Overlap 이벤트를 “실행 트리거”로 착각하여
    - 클라/서버 모두에서 처리해야 하는지 혼동 발생
- 자동 픽업 동작이 처음엔 “버그처럼” 보였으나,
    - 서버 권한 관점에서는 정상 동작임을 확인

## Fix / Improvement
- Overlap을 실행 요청이 아닌 조건/후보 신호로 재정의
- Interaction Sphere를 별도 컴포넌트로 분리하여
    - 이동 캡슐과 상호작용 로직 간 결합 제거
- 자동 픽업은 서버 Overlap 전용으로 단순화

## Next Step
- 입력 기반 픽업(E 키)과 자동 픽업의 공통 인터페이스 정리
- CanInteract 실패 사유(Enum) 반환을 통한 UI 피드백 확장
- 다수 후보 중 우선 대상 선택 로직(거리/시야각) 추가
