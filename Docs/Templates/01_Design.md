## Goal
이 티켓에서 달성하려는 설계 목표를 한 문단으로 요약

## Class / Component Responsibilities
각 클래스 / 컴포넌트의 역할과 책임 정리

## Data Flow
입력 → 처리 → 결과까지의 흐름을 단계적으로 설명

## Edge Cases
예외 상황, 경계 조건, 문제가 생길 수 있는 케이스

## Perf / GC / Tick Notes
성능, 메모리 관리, Tick 사용 여부에 대한 고려 사항

## Replication Notes (선택)
네트워크 환경에서의 처리 방식 또는 향후 고려 사항
